Administration: Create Project Template

Create Project Template

Create a template of your project as it currently stands. This can include all wiki pages, ticket types, workflows, milestone, mailing lists, membership groups and the latest revision from your Subversion repository (currently we do not offer the ability to export GIT repositories). This can then be used when a new #define project is created. Please note that this is not an appropriate means of backing up your project content - that is a very much seperate, and already automated process.

The name should uniqley identify this template.
Please provide a few words to describe the purpose of this template. For example, does it provide a template for teams implementing a SCRUM methodology or was it designed to be used by a particular project etc.

You can customise which parts of your project you export into a template. For example you might only want to export wiki pages, mailing lists and membership groups. By default all components are exported.

When a new project is created with your template, default #define data will be used if you have not exported that component in your template.
